Game loop Implementation

Here I implemented my game loop that has a constant game speed that is independent of the FPS that other machines render at. in this implementation I have a function that gets recursively called in order to complete the loop, while I also have a while loop to create a fixed Update, basically where the game can get called at an FPS I choose, where in this case I ran the game at 60 FPS. This way I can have consistent simulation across multiple machines without losing any of the graphical power that the machine can have.

Anagram Board Generation

The problem I face was how to create a board that is guaranteed to have some words that players can find. One possible idea I played around with was to create pre-generated boards, then load those into the game when the player needs a new board. that would be a time-consuming task, that wouldn't overall improve the experience of the player since there would be a limited about of board the player can choose from. So I opted to use a letter frequency chart and implement the probabilities of each word being generated. Since my game is using only English words I can generate letters based on how often they show up in the English language. This makes the likely hood of a real English word popping up to be 99%.

Project Description

Fishogram is mobile game where players need to solve for different anagram in order to raise the water level and bring their pet fish to the pip in order for them to get out the sewer system.

Tools Used

  • Libraries: Vite framework, Pixi.js as a renderer
  • Programming Language: Typescript
  • IDE: Visual Studio Code
  • Source Control: Github

What I learned

This project made it so I can have a better understanding of the input -> simulate -> render game loop format. While also allowing me to creatively solve problems that I wouldn't normally encounter. I struggled at many different points but was able to pick myself up and try out something new that lead to solutions.
