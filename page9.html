Unreleased Mobile Title

September 2023 - March 2024

Project Description

A 3D physics game where the user is tasked to fling an object into the air and create as much contact as possible. Throughout the project there was a utilization of an entity - component- system API. While also developing the tooling to use Godot as an exporter for the 3D object metadata and scene data.

Tools Used

  • Libraries: Vite framework, Rapier.js (Physics Engine), Three.js (Render) Godot (Scene Builder)
  • Programming Language: Typescript, C#
  • IDE: Visual Studio Code
  • Source Control: Github

What I learned

A project like this builds my experience in creating games in a professional setting. Most important lessons I learned were communication between engineers and not everything is about gameplay code. Throughout this project a good chunk of my time was communicating and structure my code properly to be able to push my code into main. After facing push rejection and correcting it after discussion and interaction I was able to understand the professional game making pipeline much better.
The other key take away I had from this project is that an engineer's job isn't only to program gameplay code, having to work lower level to help integrate tools such as Godot to make the 3D objects to Renderer pipeline for the artist a lot more simpler and efficient.
